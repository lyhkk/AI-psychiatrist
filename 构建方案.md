# CBT-Discover 心理辅助与多轮对抗评测系统：Vibe Coding 执行手册

## 一、 项目全局设定与架构基座 (Architecture & Setup)

**1. 核心目标** 构建一个基于 LangGraph 的多智能体心理辅助系统。系统必须彻底解耦“临床诊断”与“对话干预”，通过隐式思维链（MDP CoT）提升临床保真度，并通过引入“患者模拟器”与“信息熵PQA”实现全自动的科学量化评测 。

**2. 技术栈要求**

- **流程编排**：`langgraph`, `langchain-core` （利用其强大的状态持久化和图执行能力）。
- **数据校验与提取**：`pydantic` （强制所有非对话的Agent使用Structured Output输出严格的JSON格式）。
- **大模型接口**：统一使用通用的LLM API调用方式（支持按需切换DeepSeek、GLM或GPT-4o等）。

------

## 二、 核心数据流与图状态定义 (State Schema)

在 LangGraph 的 `StateGraph` 中定义全局共享状态，这是所有智能体交互的唯一媒介。

要求AI助手定义以下 `TypedDict`：

- `chat_history` (List): 标准的多轮对话消息列表。
- `cbt_form` (Dict): 结构化的认知行为表单。包含四个核心键：`situation`（情境）、`emotion`（情绪）、`automatic_thought`（自动思维）、`cognitive_distortion`（认知扭曲类别）。初始值均为 `null`。
- `entropy_scores` (List[float]): 用于记录对话过程中的信息熵变化。

------

## 三、 干预系统：双核智能体设计 (Therapeutic Agents)

本模块摒弃了传统的“Supervisor指导Therapist说话”的模式，采用完全解耦的双Agent协同。

### Agent 1: 临床循证追踪器 (Diagnostician Agent)

- **职责**：在每次患者输入后作为“静默节点”运行。它不生成任何对外对话，仅负责提取上下文并填补 `cbt_form` 中的空缺 。

- **System Prompt 设定**：

  > “你是一个后台临床循证追踪器。你的唯一任务是阅读最新对话，并更新患者的CBT认知评估表。
  >
  > 提取规则：
  >
  > 1. 从对话中抽取情境、情绪、自动思维。若发现符合的，填入对应字段；若信息不足，保持为 null。
  >
  > 2. 当收集到足够信息时，严格从12种标准认知扭曲中匹配并打上标签。
  >
  >    请严格输出最新的 JSON 对象。”

### Agent 2: MDP-CoT 治疗师 (Therapist Agent)

- **职责**：在 Diagnostician 更新表单后运行。拥有对话自主权，但必须在回复前进行“记忆驱动动态规划”。

- **System Prompt 设定**：

  > “你是一位专业的CBT心理咨询师。请参考当前后台表单的状态（`{cbt_form}`）。
  >
  > 在回复患者之前，你**必须**首先在 `<inner_monologue>` 标签内进行思考：
  >
  > 1. 防御评估：患者当前的情绪张力和防御水平如何？
  >
  > 2. 表单缺口分析：目前还有哪些字段是 null 需要探询？或者是否已经可以发起认知面质？
  >
  > 3. 策略规划：决定当前回合的话术方向。
  >
  >    思考完成后，在 `<response>` 标签内输出自然、温和的中文回复。切勿暴露后台表单内容。”

## 四、 评测系统：沙盘推演与临床量化引擎 (Evaluation Sandbox)

这是系统的核心创新区，用于替代传统的单轮F1测试，直接对系统的多轮临床能力进行压测。

### 1. 患者模拟器 (Stubborn Patient Agent)

- **职责**：模拟真实的、带有防御机制的患者，与 Therapist Agent 自动进行多轮对话（例如10-15轮）。

- **初始化数据源**：解析PsyQA数据集，提取你截图中展示的 `question`（标题）和 `description`（详细描述），作为模拟器的初始剧本。

- **System Prompt 设定**：

  > “你正在扮演一名寻求心理咨询的患者。
  >
  > 你的背景：`{PsyQA_question_and_description}`。
  >
  > 你的性格（高防御）：你非常固执，充满防御心。如果你感到咨询师在生硬地说教、直接给建议，或者无底线地迎合你的负面情绪（阿谀奉承），你会感到厌烦并试图抗拒。只有当咨询师展现出深刻共情并用巧妙的问题引导你时，你才会逐渐吐露真实想法。”

### 2. 信息增益计算器 (Entropy-based PQA Evaluator)

- **理论依据**：真正有效的主动提问（PQA），应该能降低系统对患者认知扭曲判断的不确定性 。
- **实现逻辑**：
  1. 编写一个轻量级的评估函数，调用LLM并要求其基于当前对话输出12种认知扭曲的**概率分布**（JSON格式，例如 `{"非黑即白": 0.6, "灾难化": 0.3...}`，总和为1）。
  2. 在 Therapist 发起提问**前**，计算一次分布的香农熵 $\mathcal{H}(P_t)$ 。
  3. 在 Patient 回答**后**，再计算一次香农熵 $\mathcal{H}(P_{t+1})$。
  4. 计算信息增益 $IG(A_t) = \mathcal{H}(P_t) - \mathcal{H}(P_{t+1})$ 。若 $IG > 0$，则记为一次高价值的苏格拉底式提问。

### 3. 全局临床保真度裁判 (Global CTRS Judge)

- **职责**：在患者模拟器与治疗师完成多轮对抗后，读取完整的对话轨迹（Transcript），基于认知疗法评定量表（CTRS）进行打分 。

- **System Prompt 设定**：

  > “你是一名资深CBT临床督导。请阅读以下完整的多轮心理咨询记录。 请根据《认知疗法评定量表 (CTRS)》的以下三个核心维度，进行 0-6 分的严格打分（0=破坏性，3=合格，6=优秀）：
  >
  > 1. 理解与共情 (Understanding)：是否准确捕捉了患者内部现实，且避免了虚假的无底线迎合？
  > 2. 人际效能 (Interpersonal Effectiveness)：是否展现了真实专业性并化解了高防御状态？
  > 3. 引导式发现 (Guided Discovery)：是否有效使用了苏格拉底提问引发认知松动，而非直接说教？ 请输出JSON，包含每个维度的 `score` 以及引述对话原句的 `justification`（判分依据）。”
